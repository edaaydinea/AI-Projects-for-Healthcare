# Contributing Guidelines

Thank you for your interest in contributing to this repository.

This repository represents a **personal project portfolio** focused on healthcare-related applications of Artificial Intelligence, Machine Learning, and Data Science. While external contributions are welcome in limited forms, the scope and ownership of the projects are intentionally preserved.

Please read the guidelines below before opening an issue or pull request.

---

## ‚úÖ Accepted Contributions

I welcome contributions in the following cases:

### 1. Minor Documentation Fixes
- Typo corrections
- Grammar or spelling fixes
- Markdown formatting improvements
- Broken link fixes

> Note: Documentation-only contributions are appreciated but **do not result in contributor or authorship credit**.

### 2. Bug Reports
- Clear bug descriptions
- Reproducible issues related to notebooks or code execution
- Environment or dependency-related issues

Please open an issue instead of a pull request for bug reports.

### 3. Suggestions and Discussions
- Conceptual suggestions
- High-level improvement ideas
- Research or methodology discussions

These should be shared via **GitHub Issues**, not direct pull requests.

---

## ‚ùå Not Accepted Contributions

The following types of contributions are **not accepted**:

- Adding names to contributor, author, or credit lists
- Refactoring or rewriting existing projects
- Adding new projects or notebooks
- Modifying research logic, methodology, or results
- Structural or architectural changes to the repository

This repository is **not intended to be a collaborative codebase**.

---

## üßæ Contributor Credit Policy

- Contributor or authorship credit is **only considered** for substantial technical or research-level contributions.
- Minor edits (documentation, formatting, grammar) are **not credited** in contributor lists.
- The repository owner retains final decision authority over attribution and credit.

---

## üìå Pull Request Policy

- Pull requests that do not align with these guidelines may be closed without merge.
- Closed pull requests do not imply negative feedback and are part of normal repository maintenance.

---

Thank you for respecting these boundaries and for your interest in this work.
